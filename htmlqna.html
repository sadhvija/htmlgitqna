GIT INTERVIEW QUESTIONS

1.What is GIT? A:GIT is defined as global information tracker.Version control system software,it is used to communicate globally.
2.Difference between git and github? A:Git is a free, open source version control tool that developers install locally on their personal computers, while GitHub is a pay-for-use online service built to run Git in the cloud.
3.What is local repository and github remote repository? A:Local repository exists in our laptop or current working directory. Remote repository exists in cloud or server.
4.What is the command used to initialize the git in your local directory? A:git init
5.What is the command used to move changes to staged area? A:git add .
6.What is the command used to commit all staged changes to local repository? A:git commit
7.Difference betweeen push and pull in git? A:Push is used to push the file from local to remote. Pull is used to pull the file from remote to local.
8.What are the ways of getting the code from github? A:There are 4 ways ,they are:
Pull,Fetch,Clone,Fork
9.Difference between all types of getting the code? A: pull copies changes from a remote repository directly into your working directory, , while git fetch does not. The git fetch command only copies changes into your local Git repo.The git pull command does both. forking - allows anybody to 'fork' the repository, basically making a local copy in their own Git repository account. They can then make changes and when finished send a 'pull request'. a clone is a copy of a repository.When you have a remote repository you issue the git clone command against its URL and you then end up with a local copy, or clone, of the repository.
